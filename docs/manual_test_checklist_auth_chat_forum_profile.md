# 手测清单：注册 -> 登录 -> 首页 -> 社区发帖 -> 点击头像聊天 -> 个人中心改密码

## 一、测试前准备
1. 启动后端（默认 `http://localhost:8080`）。
2. 启动前端（Android 模拟器建议 API 基址使用 `http://10.0.2.2:8080`）。
3. 确认数据库连接信息有效：`localhost:3308`，库名 `20260419_ai_intelligent_waste_sorting_identification_app`。

## 二、用例 1：注册后不能直接进入系统，必须登录
1. 打开 App，默认进入登录页。
2. 点击 “No account? Register first” 进入注册模式。
3. 输入新账号信息（建议邮箱带时间戳避免重复）。
4. 点击 “Register”。
5. 预期：
   - 显示注册成功提示。
   - 页面切回登录模式。
   - 不会直接进入首页。

## 三、用例 2：登录后才能访问业务功能
1. 使用刚注册的邮箱+密码点击 “Login”。
2. 预期：
   - 成功进入主界面（底部有 Home/Classify/Rewards/Forum/Messages/Profile）。
3. 关闭 App 重开，重新登录后进入任意业务页。
4. 预期：
   - 不出现 401 未授权错误。
   - 分类、社区、消息、个人中心均可正常请求。

## 四、用例 3：首页与基础数据访问
1. 进入 Home 页。
2. 预期：
   - 页面正常展示欢迎与卡片内容。
   - 天气组件存在（若高德英国无实时数据，显示“无数据/失败提示”也算符合预期）。

## 五、用例 4：社区发帖
1. 进入 Forum 页，点击右上角新增按钮。
2. 输入标题、正文、标签，点击发布。
3. 预期：
   - 新帖子出现在列表顶部。
   - 作者为当前登录用户。

## 六、用例 5：点击头像进入聊天并发送消息
1. 在 Forum 页点击其他用户帖子头像。
2. 预期：
   - 成功打开聊天页。
   - 会创建或复用与该用户的 direct conversation。
3. 在聊天页发送一条文本消息。
4. 发送一张图片消息（从相册选择）。
5. 返回 Messages 页查看会话列表。
6. 预期：
   - 会话存在，预览内容更新。
   - 重新进入会话后可看到刚发送的文字和图片。

## 七、用例 6：个人中心修改密码
1. 进入 Profile 页，点击 “Password”。
2. 输入当前密码和新密码并确认提交。
3. 预期：
   - 提示密码修改成功。
4. 退出到登录页，使用旧密码登录。
5. 预期：
   - 登录失败（提示账号密码错误）。
6. 使用新密码登录。
7. 预期：
   - 登录成功。

## 八、回归检查（建议）
1. 改密后再次进入 Forum 发帖。
2. 点击头像进入聊天并发送文本。
3. 进入 Profile 修改昵称/邮箱/城市并保存。
4. 预期：
   - 所有操作正常，未引入新 401 或参数错乱问题。
